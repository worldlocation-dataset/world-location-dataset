Static API-style Endpoints

Although this project is a static website, the files can be consumed like a simple API. All endpoints are just static JSON files.

Countries

  • /data/countries.json – Array of country objects
  • /data/countries.csv – Same data in CSV format

Cities (by country)

Each country has its own file based on the ISO 3166-1 alpha-2 code:

  • /data/cities/IN.json – All cities in India
  • /data/cities/US.json – All cities in the United States
  • /data/cities/GB.json – All cities in the United Kingdom
  • … and so on for each country.

Usage examples

JavaScript

const iso2 = 'IN';
fetch(`/data/cities/${iso2}.json`)
  .then(res => res.json())
  .then(cities => {
    console.log(`Cities in ${iso2}:`, cities.length);
  });

Python

import requests

iso2 = "US"
resp = requests.get(f"https://your-domain.com/data/cities/{iso2}.json")
cities = resp.json()
print("Cities in US:", len(cities))

Countries Schema

Field Type Description Example
iso2 string 2-letter ISO 3166-1 alpha-2 country code "IN"
iso3 string 3-letter ISO 3166-1 alpha-3 country code "IND"
name string Country name "India"
capital string Capital city name "New Delhi"
region string Geographical region "Asia"
subregion string Sub-region "Southern Asia"
latitude number Latitude of capital 28.6139
longitude number Longitude of capital 77.2090

Cities Schema

Field Type Description Example
name string City name "Chennai"
country_iso2 string 2-letter country code linked to iso2 in countries "IN"
state string State or region name, if available "Tamil Nadu"
is_capital boolean True if this city is a capital city false
latitude number Latitude of the city 13.0827
longitude number Longitude of the city 80.2707
population number|null Population (if known), or null 7090000
